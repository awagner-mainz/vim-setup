AsNeeded.vim : Loads plugins as-needed (function/map/commands)  (autoload)

 script karma  Rating 26/8, Downloaded by 653

created by
Charles Campbell
 
script type
utility
 
description
The AsNeeded plugin loads plugins from the .vim/AsNeeded directory only as they're needed.

Whenever a plugin attempts to execute a function that hasn't been loaded yet, AsNeeded will find the script in the AsNeeded directory, load the script, and execute the function.

Whenever you want to execute a command or map that's not been loaded yet, you can use

:AN map
:AN command [arguments]

and AsNeeded will locate the map or command, load the script, and make the map or command available for your subsequent use.  Alternatively, you can use

:ANX map
:ANX command [arguments]

and AsNeeded will do what :AN would do plus attempt to execute the map or command.

For even faster loading of your desired function, map, or command, you can (as of v6) use

:MakeANtags

This command will create a <ANtags> file in your AsNeeded subdirectory which basically lists your maps, functions, and commands and couples them with their associated source file.  When you then use :AN, :AsNeeded, or :ANX, AsNeeded will use that file to quickly load the associated source file.

install details
1. In your <.vimrc>: to enable plugins,
   set nocp
   if version >= 600
     filetype plugin on
   endif

2. Make a <.vim/AsNeeded> directory:
   cd .vim
   mkdir AsNeeded

3. Put whatever plugins you want into the AsNeeded directory

4. Put <AsNeeded.vim> itself into the .vim/plugin directory
   Note - the next time you bring up vim, the HelpExtractor
   embedded in AsNeeded will transparently enable AsNeeded's
   help.
